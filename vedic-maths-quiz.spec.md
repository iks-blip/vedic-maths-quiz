# Vedic Maths Challenge --- Technical Specification (Phase 1)

## Philosophy

This project follows a **feature-first development approach**.

Rules:

1.  Think in **features**, not products.
2.  No feature should be built without **deep questions answered
    first**.
3.  Build **Feature 1 → Test → Feature 2 → Test → Continue**.
4.  Never stack features on an **untested foundation**.
5.  Each feature must include:
    -   Acceptance criteria
    -   Unit tests
    -   Integration tests

Goal: Spend **maximum time planning**, minimum time debugging.

------------------------------------------------------------------------

# Phase 1 --- Core Goal

Build a **competitive Vedic Maths quiz platform** for college students
and faculty.

Primary objectives:

-   Fast mental math competition
-   Live leaderboard engagement
-   Certificate eligibility for strong performers

Success metrics:

-   \~500 quiz attempts
-   \~50 certificates issued
-   Active leaderboard participation

# Question Bank Structure

Total questions: **75**

Difficulty distribution:

  Difficulty   Count
  ------------ -------
  Easy         25
  Medium       25
  Hard         25

Each quiz attempt receives:

8 Easy\
9 Medium\
8 Hard

Questions are: - randomly selected - randomly ordered

------------------------------------------------------------------------

# Quiz Flow (Happy Path)

1.  User visits quiz website
2.  User clicks **Start Quiz**
3.  User enters:
    -   Name
    -   Email
4.  System generates **25 random questions**
5.  Quiz begins
6.  Questions appear **one at a time**
7.  User selects answer
8.  Next question appears
9.  After 25 questions → submit
10. Score calculated
11. Score shown to user
12. Leaderboard updates live

Certificates are generated **after the event ends**.

------------------------------------------------------------------------

# Quiz Rules

### Navigation

-   One question at a time
-   Cannot go back to previous questions
-   Timer starts when question appears
-   Copy/paste disabled
-   Tab switching detected

### Tab Switch Policy

First tab switch → Warning\
Second tab switch → Auto submit

------------------------------------------------------------------------

# Timer Rules

  Property             Value
  -------------------- ----------------------------
  Global quiz timer    20 minutes
  Timer start          When quiz begins
  Timer end behavior   One-question grace allowed

If timer hits zero:

User may finish the **current question only**, then quiz auto-submits.

------------------------------------------------------------------------

# Resume Rules

If user:

-   closes tab
-   refreshes page
-   leaves the quiz

Then:

Attempt is void\
User must start over\
Previous attempt deleted

------------------------------------------------------------------------

# Internet Disconnect Rule

Grace window:

Disconnect \< 60 seconds → resume quiz\
Disconnect \> 60 seconds → auto submit

Timer **continues running** during disconnect.

------------------------------------------------------------------------

# Scoring System

Points awarded **only for correct answers**.

  Time to answer   Points
  ---------------- --------
  ≤10 seconds      10
  11--15 seconds   8
  16--20 seconds   6
  \>20 seconds     4
  Wrong answer     0

Maximum possible score:

25 questions × 10 = 250 points

------------------------------------------------------------------------

# Leaderboard

Leaderboard behavior:

-   Updates **after every submission**
-   Visible **during the event**
-   Shows:

  Rank   Name   Score
  ------ ------ -------

Leaderboard sorted by:

Score DESC\
Submission time ASC

------------------------------------------------------------------------

# Certificate Eligibility

Certificates are generated **after the event ends**.

Eligibility rule:

Score ≥ 180

Certificates generated by a **batch script** the next day.

Each certificate contains:

-   Name
-   Score
-   Certificate ID
-   Date issued
